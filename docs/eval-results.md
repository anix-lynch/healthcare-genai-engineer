## Per-metric meaning

```
any_hit_rate          fraction of queries that returned ≥ 1 hit.
                       0.65 = 13/20 queries got at least one BM25 hit.
                       Misses are queries whose tokens don't appear
                       verbatim in the 497-row corpus (dense path
                       would help; see roadmap).

faithfulness_avg      custom-proxy: 1.0 if answer cited only real
                       source_ids + no dropped-citation warnings.
                       0.65 average reflects same misses as any_hit
                       (no hits → faithfulness=0 by construction).

condition_relevance   fraction of returned citations whose snippet
                       contained the query's expected condition keyword.
                       0.567 = retrieval is hitting RELATED conditions
                       (e.g., "diabetes" query returns endocrine-adjacent
                       cases) but not always exact-keyword precision.

p95 latency           5 ms — BM25 over an in-memory 497-row corpus
                       is microseconds. Real BM25 over 55K rows ≈
                       50-200ms; dense path adds the encoder latency.
```

## Regression gate

`evaluation/regression_gate.py` tracks 4 metrics with these tolerances:

```
any_hit_rate         drop > 0.05  → fail
faithfulness_avg     drop > 0.05  → fail
relevance_avg        drop > 0.10  → fail
p95_latency_ms       rise > 200ms → fail
```

Current baseline (committed at `evaluation/baseline.json`):
```
any_hit_rate         0.650
faithfulness_avg     0.650
condition_relevance  0.567
p95_latency_ms       5
```

Future PRs that move these past tolerance will fail `make gate`.
