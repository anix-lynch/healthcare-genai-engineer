# Senior Gaps

This repo is already past the basic demo stage. The remaining work is the
smallest set of visible gaps that keep it from reading like a strong $200k+
AI engineer repo.

## Pragmatic backlog

1. Retrieval proof, not just a baseline snapshot.
2. Real trace artifact and debug playbook.
3. Fail closed on unsupported citations and adversarial safety cases.
4. Buyer-grade operating model in the top-level story.

## Stricter split

If you want the more literal reviewer split, issue 1 becomes:

1. Retrieval rationale and before/after proof.
2. Eval history and trend visibility.

That makes the backlog 5 instead of 4. I do not think the extra split is
necessary yet, but it is defensible.

## Why this exists

A senior reviewer should be able to open one file and see what still blocks
the repo from feeling flagship-level, without guessing which comments were
already addressed in code.
